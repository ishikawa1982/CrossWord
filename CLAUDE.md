# CrossWord 開発メモ

## バージョン運用（必須ルール）
- **コードや挙動に変更を加えたら、必ずバージョンを上げてからコミットする。**
- バージョンはルートの `package.json` の `version` フィールド（例: `0.2.1`）。
- このバージョンはビルド時に `__APP_VERSION__` として埋め込まれ、画面右下のフッターに `v0.2.1` のように表示される（`client/vite.config.ts` で `package.json` を読み込み）。
- 増分の目安（セマンティックバージョニング）:
  - パッチ（`0.2.1` → `0.2.2`）: バグ修正・小さな調整
  - マイナー（`0.2.1` → `0.3.0`）: 機能追加
  - メジャー（`0.x` → `1.0.0`）: 互換性を壊す大きな変更
- 変更内容に応じて適切な桁を上げること。コミットメッセージにもバージョンが分かるようにすると良い。
- **バージョンを上げたら、`CHANGELOG.md` の先頭にも同じバージョンの変更点を追記する。**

## 構成（モノレポ / npm workspaces）
- `shared/` — 型定義・クロスワード生成・ゲームロジック・正規化・単語データ（クライアント/サーバ共有）
- `server/` — Express + Socket.IO（対戦モードのサーバ権威。`client/dist` があれば同一オリジンで配信）
- `client/` — React + Vite（ソロモード＋対戦モード）

## デプロイ
- `render.yaml` による Render の無料プラン（`plan: free`）で、サーバが画面も配信する単一サービス構成。
- `main` への push で Render が自動再ビルド・再デプロイ。

## テスト・ビルド
- `npm test` … shared のユニットテスト（vitest）
- `npm run build` … 全ワークスペースの型チェック＋クライアント本番ビルド
- 変更後はテストとビルドが通ることを確認してからコミットする。
